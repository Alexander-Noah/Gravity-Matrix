# Gravity-Matrix 项目讲解录屏脚本

> 建议视频时长：6 到 10 分钟。
> 录制方式：一边操作网页，一边按下面“讲解词”说明。

## 0. 开场介绍

### 操作

打开项目首页或登录页。

### 讲解词

大家好，我展示的是 Gravity-Matrix，一个 AI 小说转剧本工具。这个项目的目标是把 3 章以上的小说文本转换成结构化剧本 YAML，让作者可以快速得到一份可编辑、可校验、可导出的剧本初稿。

整个系统包含前端和后端两部分。前端使用 Vue 3 和 Vite，提供登录注册、工作台、模板中心、剧本库和帮助文档。后端使用 FastAPI、SQLAlchemy 和 SQLite，负责小说导入、AI 解析、剧本生成、YAML 校验、质量诊断和多格式导出。

接下来我会按真实使用流程演示：登录、导入小说、AI 解析、选择模板、生成剧本、编辑校验、预览导出，以及剧本库管理。

## 1. 登录注册页面

### 操作

展示 `/auth` 登录注册页面。可以注册一个账号，也可以直接登录已有账号。

### 讲解词

首先进入系统会看到登录注册页。这里实现了基础账号体系，包括注册、登录、获取当前用户和退出登录。

系统默认入口是登录页，并且做了路由拦截。如果用户没有完成本次登录，直接在地址栏输入工作台、模板中心、剧本库或帮助文档地址，也会被拦回登录页，避免绕过登录直接进入主页。

我这里登录后进入工作台。

## 2. 工作台整体流程

### 操作

登录后进入 `/workbench`，展示顶部标题、侧边栏、流程步骤。

### 讲解词

登录后进入小说转剧本工作台。这里是项目的核心页面，流程是从导入小说开始，依次完成 AI 解析、剧本生成、在线编辑和导出。

页面上方有流程提示，左侧是主要导航，包括工作台、模板中心、剧本库和帮助文档。这个设计是为了让用户在长流程中始终知道自己当前处在哪一步。

## 3. 导入小说文本

### 操作

在导入页粘贴一段至少 3 章的小说，或上传 TXT 文件。点击下一步。

### 讲解词

第一步是导入小说。系统支持直接粘贴文本，也支持上传一个或多个 TXT 文件。多文件会按文件名顺序合并，如果文件本身没有章节标题，前端会自动补全章节标题。

后端提供了导入预检接口，会识别“第1章”“第一章”“Chapter 1”等章节格式，并返回章节数量、字数、正文摘录和是否满足创建项目条件。

这个项目要求至少 3 章小说才能创建项目，这样后续 AI 分析和剧本生成才有足够上下文。

## 4. AI 解析页面

### 操作

进入 AI 解析页，展示人物、场景、剧情事件、关系、对白线索等内容。

### 讲解词

进入第二步后，系统会启动 AI 解析任务。后端会把小说章节整理成结构化素材，提取主要人物、地点、章节摘要、剧情事件、主题和冲突。

这里可以看到人物信息、剧情节点、人物关系和对白线索。这些内容不是单纯展示用的，它们会作为后续剧本生成的上下文。

任务接口会记录进度和状态，前端通过轮询任务状态展示当前步骤，比如正在读取章节、正在分析人物和场景、正在整理结果。

如果没有配置真实大模型 API Key，系统也会使用本地确定性演示逻辑，保证项目在本地环境可以完整跑通。

## 5. 模板和生成设置

### 操作

点击“下一步”或“生成剧本”，打开生成设置弹窗，展示模板、剧本类型、改编偏好。

### 讲解词

生成剧本前，用户可以选择剧本模板和生成偏好。目前系统提供 5 种模板：影视剧、短剧、话剧、分镜剧本和广播剧。

不同模板会影响后端生成规则。比如短剧模板会更强调开场冲突和反转，分镜模板会更强调镜头化描述，广播剧模板会更强调声音和环境音。

这些生成设置会保存到项目中，并参与后端 prompt 和兜底生成逻辑，不是只存在前端的展示项。

## 6. 生成 YAML 剧本

### 操作

确认生成设置，等待生成完成，进入剧本编辑页。

### 讲解词

点击确认后，系统会启动剧本生成任务。后端会根据小说章节、AI 解析结果和模板设置生成结构化 YAML 剧本。

生成结果包含剧本元信息、人物、地点、章节、场景、出场人物、场景梗概、动作说明和对白。

YAML 是这个项目的核心输出格式。相比普通文本，YAML 结构更清晰，方便后续编辑、校验、诊断、导出和二次处理。

## 7. YAML 编辑和结构导航

### 操作

展示剧本编辑页，点击左侧章节/场景导航，展示 YAML 定位。尝试手动修改一行 YAML。

### 讲解词

这里是剧本编辑页。左侧有剧本结构导航，右侧是 YAML 编辑区和预览信息。

用户可以点击章节或场景，系统会自动定位到对应的 YAML 行。黑色编辑框支持手动编辑，修改后会进入自动保存流程。

这个功能的意义是：AI 生成的剧本不是最终稿，而是一个可以继续打磨的初稿。作者可以直接修改对白、场景标题、动作说明和人物信息。

## 8. Schema 校验和质量诊断

### 操作

点击“校验 YAML”或展示校验结果区域。

### 讲解词

编辑完成后，可以进行 Schema 校验。Schema 校验会检查 YAML 的字段结构、必填项、章节、场景、人物引用和地点引用是否符合项目定义。

除了结构校验，后端还提供质量诊断，会输出分数、等级、统计摘要、优点和问题列表。比如场景数量是否足够、对白是否过少、人物引用是否正确、章节摘要是否完整等。

这一步可以帮助用户发现 AI 初稿中的结构问题，让剧本更容易进入后续修改和交付。

## 9. 新增场景

### 操作

点击新增场景按钮，展示弹窗，填写或简单说明字段。

### 讲解词

如果用户觉得某一章缺少关键场景，可以通过新增场景功能补充内容。新增场景会写入后端，并重新同步到当前 YAML 中。

这个功能体现了系统不是一次性生成工具，而是支持生成后继续编辑和扩展。

## 10. 完整预览

### 操作

点击进入完整预览页，展示标准剧本文本、场景概览、统计信息。

### 讲解词

编辑完成后，可以进入完整预览页。这里会把 YAML 转换成更适合阅读的剧本文本形式，展示场景、出场人物、动作说明和对白。

右侧有场景概览，用户可以快速浏览剧本结构。顶部也会显示场景数、角色数和对白数，帮助用户快速判断剧本规模。

## 11. 多格式导出

### 操作

依次展示导出 YAML、TXT、Markdown、PDF 的按钮。可以点击其中一个导出。

### 讲解词

系统支持多格式导出：

YAML 适合继续编辑和程序读取；TXT 适合纯文本阅读；Markdown 适合带标题层级的阅读稿；PDF 用于生成可下载的文件。

这里的 PDF 导出已经改成后端生成并下载文件，不再调用浏览器打印窗口，用户体验更接近真实导出。

## 12. 剧本库

### 操作

进入 `/library`，展示剧本卡片、本地素材、导出、编辑、预览、删除等操作。

### 讲解词

接下来是剧本库页面。剧本库会整合两类数据：一类是用户已经生成的剧本，另一类是后端本地素材库中的小说素材。

对已生成剧本，用户可以继续编辑、预览、导出、重命名、克隆或删除。对本地素材，用户可以一键导入工作台，继续进行 AI 解析和剧本生成。

剧本库的数据来自后端真实接口，不再依赖前端 mock 数据。

## 13. 模板中心

### 操作

进入 `/templates`，展示 5 种模板，切换或选择一个模板。

### 讲解词

模板中心用于管理剧本生成规则。每个模板都有适用场景、目标格式、后端规则、字段说明和 YAML 示例。

用户可以设置默认模板。默认模板会保存到后端，后续进入个人中心和生成设置时都可以读取到真实的模板状态。

## 14. 帮助文档

### 操作

进入 `/help`，展示帮助文档和结构导航高亮。

### 讲解词

帮助文档页面用于说明系统流程、Schema 字段和设计原因。由于文档内容比较长，页面提供结构导航，并且会根据当前阅读位置高亮导航项，方便用户知道自己正在看哪一部分。

这对于比赛展示和后续用户使用都比较重要，因为 YAML Schema 不是普通用户天然熟悉的格式，需要有清晰说明。

## 15. 个人中心

### 操作

点击右上角用户入口，打开个人中心。

### 讲解词

个人中心展示当前用户和工作区数据，包括当前项目、项目进度、流程阶段、模板、剧本状态、剧本库数量和 Schema 状态。

这部分数据后端提供了聚合接口，会根据真实项目、模板和剧本状态生成，而不是前端写死。

## 16. 后端接口和技术亮点总结

### 操作

可以打开 API 文档 `/docs`，或切回项目页面总结。

### 讲解词

最后简单总结后端能力。后端主要提供认证、导入预检、项目管理、AI 解析任务、剧本生成任务、模板设置、剧本编辑、Schema 校验、质量诊断和多格式导出。

任务系统会记录排队、运行、成功和失败状态；工作台聚合接口可以一次返回项目、流程、解析、剧本和诊断数据；剧本库接口会聚合已生成剧本和本地素材。

这个项目的重点不是只生成一段文本，而是形成一个完整的小说改编工作流，把小说文本转成可编辑、可校验、可导出的结构化剧本资产。

## 17. 结尾总结

### 操作

回到工作台或预览页，停留在最终剧本效果。

### 讲解词

以上就是 Gravity-Matrix 的完整功能演示。它覆盖了从小说导入、AI 解析、模板化剧本生成，到 YAML 编辑、质量校验、多格式导出和剧本库管理的完整链路。

项目可以帮助创作者快速获得剧本初稿，并把后续修改工作建立在结构化数据之上。后续还可以继续扩展真实用户项目隔离、更专业的 PDF 排版、更高质量的生成策略和更完整的权限体系。

我的演示到这里结束，谢谢观看。

## 录制提示

- 录制前先启动后端和前端。
- 提前准备一段至少 3 章的小说文本，避免现场粘贴出错。
- 如果大模型 API 不稳定，可以说明系统支持本地确定性兜底逻辑。
- 演示时重点展示“从导入到导出”的闭环，不要只停留在页面介绍。
- 每一步操作后停顿 1 到 2 秒，方便观众看清页面变化。
- 如果时间有限，重点讲：导入预检、AI 解析、模板生成、YAML 编辑、Schema 校验、导出、剧本库。
