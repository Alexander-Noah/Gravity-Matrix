# Gravity-Matrix 功能说明

## 1. 项目概述

Gravity-Matrix 是一个 AI 小说转剧本系统。用户导入小说后，系统会识别章节、解析人物和剧情元素，再根据选定的生成方式输出结构化 YAML 剧本。用户可以继续编辑、校验、预览和导出。

系统由两部分组成：

- 前端：Vue 3 + Vite + Element Plus，提供工作台、模板中心、剧本库、帮助文档和个人中心。
- 后端：FastAPI + SQLAlchemy + SQLite/MySQL，负责认证、项目管理、AI 调用、YAML 校验、质量诊断和导出。

## 2. 用户账号

系统提供基础账号能力：

- 注册：用户名、邮箱、密码。
- 登录：邮箱和密码。
- 会话：前端保存 JWT token。
- 当前用户：通过 `/auth/me` 获取登录信息。
- 个人中心：通过 `/profile/summary` 展示工作区、项目、模板、剧本库和校验状态。

未登录用户访问工作台、模板中心、剧本库或帮助文档时，会被重定向到 `/auth`。

## 3. 小说导入

导入页支持：

- 粘贴小说正文。
- 上传 TXT 文件。
- 本地预览章节。
- 后端导入预检。

导入预检会返回：

- 标题和作者。
- 章节数量。
- 每章标题、正文、摘录和字数。
- 是否满足创建项目条件。
- 本地预处理得到的人物、地点、主题和冲突候选。

默认要求至少 3 章，避免上下文过少导致剧本生成质量不稳定。

## 4. 结构化解析

解析任务负责把小说文本整理为剧本生成所需的结构化素材。

当前解析策略：

- 按 `LLM_CHUNK_SIZE` 切分文本。
- 每个 chunk 只调用一次 LLM。
- 一次返回 `characters`、`locations`、`organizations`、`events`、`dialogues`、`conflicts`。
- 后端合并去重。
- 单个 chunk 失败不会让整个任务失败，会记录到 `failed_chunks`。
- 对 `chunk_text + model + prompt_version` 计算 hash，命中缓存时直接复用。

这样可以降低 token 成本、减少重复调用，也能避免模型输出过长导致 JSON 截断。

## 5. 剧本生成

剧本生成采用分阶段生成，避免让模型一次性输出完整大 JSON。

生成流程：

1. 生成 metadata、人物、地点、组织等基础信息。
2. 按章节逐章生成 scenes。
3. 后端用 Python 合并成最终 script JSON。
4. 转换为 YAML 并保存到项目。

每次 LLM 调用只要求返回小 JSON。返回内容会经过 safe JSON 解析，包括：

- 去掉 Markdown 代码块。
- 提取第一个 `{` 到最后一个 `}`。
- `json.loads` 失败时记录完整错误。
- JSON 解析失败时尝试要求模型只修复 JSON 格式。

## 6. 模板中心

模板中心面向普通用户，页面名称是“选择剧本生成方式”。

支持的生成方式：

- 生成影视剧剧本。
- 生成短剧脚本。
- 生成分镜脚本。
- 生成广播剧脚本。
- 生成舞台话剧。

卡片展示用户关心的信息：

- 适合什么场景。
- 生成出来是什么效果。
- 成本、速度、质量标签。
- 推荐用途。
- 生成示例。

Schema 信息保留在“更多 / 查看 Schema”里，不直接堆在卡片主体。

## 7. YAML 编辑器

生成后的 YAML 会进入可编辑工作区。

能力包括：

- 深色 YAML 编辑区。
- `yamlContent` 双向绑定。
- 用户修改后校验当前输入内容。
- 保存当前编辑后的 YAML。
- 复制和下载使用编辑后的 YAML。
- 左侧章节导航和右侧校验结果保持独立。

YAML 是项目的核心交付格式，既方便人编辑，也方便程序校验和导出。

## 8. 校验和质量诊断

校验分两层：

### 格式校验

检查 YAML 是否可解析，以及是否符合剧本结构要求：

- 顶层必须包含 `script`。
- metadata、characters、locations、chapters 等字段结构正确。
- 每个章节应包含场景。
- 场景引用的人物和地点应存在。

### 质量诊断

根据剧本内容生成质量反馈：

- 分数。
- 等级。
- 章节数、场景数、对白数。
- 优点。
- 建议优化项。
- 需要修正的问题。

用户界面中使用“格式正常”“格式需修正”“质量良好”“建议优化”“需要修正”等文案，减少技术感。

## 9. 完整预览和导出

完整预览页把 YAML 转换成更适合阅读的标准剧本文本。

支持导出：

- YAML：保留结构化源文件。
- TXT：纯文本剧本。
- Markdown：带标题层级的文档。
- PDF：适合分享和交付。

导出失败会使用统一的 Element Plus 消息提示，不使用浏览器原生弹窗。

## 10. 剧本库

剧本库用于管理草稿和版本。

卡片展示：

- 剧本标题。
- 剧本类型。
- 状态：草稿中、已完成、需修正。
- 来源小说。
- 最后编辑时间。
- 章节数、场景数、对白数。
- 格式状态。
- 质量状态。

操作包括：

- 继续编辑。
- 查看预览。
- 导出。
- 重命名。
- 复制一份。
- 重新生成。
- 移动到回收站。
- 查看生成日志。

## 11. 回收站

项目删除后进入回收站，支持：

- 查看删除记录。
- 恢复项目。
- 清空回收站。

前端也保留本地删除历史提示，但真实项目恢复以后端回收站接口为准。

## 12. AI 成本和速度优化

当前已采用：

- chunk 单次调用。
- LLM 缓存。
- `LLM_MAX_CONCURRENCY` 控制并发，默认 2。
- `LLM_MAX_RETRIES` 控制重试，默认 0。
- 429 和 5xx 最多重试 1 次。
- 401、402、403 不重试。
- 402 返回余额不足提示。

这些设置适合 DeepSeek/OpenAI-compatible 场景，能兼顾成本、速度和稳定性。

## 13. 兼容能力

项目保留早期 novel-to-yaml 接口和数据结构，但主流程已经迁移到 `/workbench`、`/projects`、`/parse/tasks` 和剧本库。

新功能开发优先使用当前工作台流程。
